Magic Beams

A small yet stylish assortment of (hopefully) easy to use utilities for the modern Erlang/OTP system

hotbeam

hotbeam handles the reloading of binary modules. optionally extracts source file location and monitors for changes in order to recompile and reload resulting beam

thunderbeam

thunderbeam will kill off random processes under the guises of testing supervisory structures

shellbeam

shellbeam provides a simple OTP behaviour allowing one to easily define bespoke REPL shells for runtime diagnostics

this optionally launches a ssh server allowing direct access to the REPL shell on a runtime system

ease of integration

The magicbeam application may be included as a rebar depdendency when building an OTP release. In addition, it may be injected into a running node. Handy when testing third party software!
